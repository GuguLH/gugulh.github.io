LangChain：LLM记忆功能从原理到最佳实践

2025年12月28日 · 阅读需 19 分钟
GuguLH
Back-end engineers

前言​

本篇文章将深入探讨LLM（大语言模型）记忆功能的实现原理，介绍常见的记忆模式，并对比LangChain 0.3x版本与1.2版本的记忆实现方式。

1. LLM实现记忆功能的思路​

LLM本身是无状态的——每次调用都是独立的，模型不会"记住"之前的对话。要实现多轮对话的记忆功能，核心思路是在每次请求时将历史对话作为上下文传递给模型。

这个流程涉及三个关键步骤：

  1. 加载历史记忆：从存储中读取之前的对话记录
  2. 组装Prompt：将历史对话与当前问题组合成完整的输入
  3. 保存对话：将当前轮次的问答保存到记忆中

2. 常见的记忆模式​

根据不同的使用场景和需求，LangChain提供了多种记忆模式。下面逐一介绍。

2.1 缓冲记忆（ConversationBufferMemory）​

最简单的记忆模式，保存所有历史对话。

特点：

  • 保留完整的对话历史
  • 随着对话增长，token消耗线性增加
  • 适合短对话场景

2.2 缓冲窗口记忆（ConversationBufferWindowMemory）​

只保留最近k轮对话，超出部分自动丢弃。

特点：

  • 固定token消耗
  • 会丢失早期对话信息
  • 适合只需要近期上下文的场景

2.3 令牌缓冲记忆（ConversationTokenBufferMemory）​

基于token数量限制，动态保留尽可能多的对话。

特点：

  • 更精细的token控制
  • 自动优化可保留的对话数量
  • 适合需要最大化利用token预算的场景

2.4 摘要总结记忆（ConversationSummaryMemory）​

使用LLM对历史对话生成摘要，用摘要替代完整对话。

特点：

  • 大幅减少token消耗
  • 可能丢失对话细节
  • 需要额外的LLM调用来生成摘要
  • 适合长对话场景

2.5 摘要缓冲混合记忆（ConversationSummaryBufferMemory）​

结合摘要和缓冲的优点，当对话超出token限制时，将旧对话转为摘要。

特点：

  • 平衡了信息保留和token效率
  • 保留近期完整对话，远期对话以摘要形式存在
  • 推荐用于大多数长对话场景

2.6 向量存储库记忆（VectorStoreMemory）​

将对话存入向量数据库，通过语义相似度检索相关历史。

特点：

  • 可扩展性好，适合超长对话历史
  • 基于语义相关性检索，而非时间顺序
  • 适合知识库问答、长期记忆场景

3. LangChain 0.3x的记忆模块示例​

LangChain 0.3x版本提供了内置的记忆模块（在1.0版本即将废弃），位于langchain_classic.memory包中。

3.1 缓冲窗口记忆示例​

import os
from operator import itemgetter

from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_classic.memory import ConversationBufferWindowMemory
from langchain_core.runnables import RunnablePassthrough, RunnableLambda
from langchain_openai import ChatOpenAI

# 1 创建提示模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的聊天机器人,请根据对应的上下文回复用户问题"),
    MessagesPlaceholder("history"),  # 历史消息占位符
    ("human", "{query}")
])

# 2 创建记忆组件 - 只保留最近2轮对话
memory = ConversationBufferWindowMemory(
    k=2,                    # 保留最近2轮
    return_messages=True,   # 返回消息对象列表
    input_key="query",      # 指定用户输入的键名
    output_key="output",    # 指定AI输出的键名
)

# 3 创建大语言模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

# 4 构建链 - 通过RunnablePassthrough.assign注入历史
chain = RunnablePassthrough.assign(
    history=RunnableLambda(memory.load_memory_variables) | itemgetter("history")
) | prompt | llm | StrOutputParser()

# 5 对话循环
while True:
    query = input("Human: ")
    if query == "quit":
        exit(0)

    chain_input = {"query": query}
    response = chain.stream(chain_input)

    print("AI: ", flush=True, end="")
    output = ""
    for chunk in response:
        output += chunk
        print(chunk, flush=True, end="")

    # 手动保存对话到记忆
    memory.save_context(chain_input, {"output": output})
    print("")

3.2 摘要缓冲混合记忆示例​

import os

    print("")

3.3 0.3x版本记忆模块的问题​

  1. 手动管理繁琐：需要手动调用load_memory_variables和save_context
  2. 与LCEL集成不够优雅：需要使用RunnablePassthrough.assign和RunnableLambda包装
  3. 官方宣布废弃：将在LangChain 1.0版本中移除

4. LangChain 1.2版本官方推荐的记忆方式​

LangChain 1.2版本对记忆系统进行了重大重构，引入了短期记忆（Short-term Memory）和长期记忆（Long-term Memory）两种核心机制。这两种记忆方式分别解决不同场景下的记忆需求。

4.1 短期记忆（Short-term Memory）​

短期记忆让应用能够记住单次会话（Thread）内的交互历史。这是最常见的记忆形式——对话历史。

4.1.1 核心概念：Checkpointer​

短期记忆通过Checkpointer实现，它会自动保存Agent的状态（包括对话历史），使得会话可以在任何时候恢复。

关键特性：

  • 线程级别隔离：每个thread_id对应独立的会话历史
  • 状态持久化：自动保存和恢复Agent状态
  • 支持多种存储后端：内存、PostgreSQL、SQLite等

4.1.2 基本使用示例​

from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver

# 创建带短期记忆的Agent
agent = create_agent(
    "gpt-4",
    tools=[get_user_info],
    checkpointer=InMemorySaver(),  # 添加Checkpointer
)

# 通过thread_id区分不同会话
agent.invoke(
    {"messages": [{"role": "user", "content": "Hi! My name is Bob."}]},
    {"configurable": {"thread_id": "session_001"}},  # 指定线程ID
)

4.1.3 生产环境配置​

在生产环境中，推荐使用数据库支持的Checkpointer：

from langchain.agents import create_agent
from langgraph.checkpoint.postgres import PostgresSaver

DB_URI = "postgresql://postgres:postgres@localhost:5442/postgres?sslmode=disable"

with PostgresSaver.from_conn_string(DB_URI) as checkpointer:
    checkpointer.setup()  # 自动创建数据库表
    agent = create_agent(
        "gpt-4",
        tools=[get_user_info],
        checkpointer=checkpointer,
    )

4.1.4 长对话处理策略​

当对话过长超出LLM上下文窗口时，LangChain提供了多种处理策略：

（1）修剪消息（Trim Messages）

保留最近的N条消息，删除较早的：

from langchain.messages import RemoveMessage
from langgraph.graph.message import REMOVE_ALL_MESSAGES
from langchain.agents.middleware import before_model

@before_model
def trim_messages(state, runtime):
    """保留最近几条消息以适配上下文窗口"""
    messages = state["messages"]

    if len(messages) <= 3:
        return None  # 无需处理

    first_msg = messages[0]
    recent_messages = messages[-3:] if len(messages) % 2 == 0 else messages[-4:]
    new_messages = [first_msg] + recent_messages

    return {
        "messages": [
            RemoveMessage(id=REMOVE_ALL_MESSAGES),
            *new_messages
        ]
    }

（2）删除消息（Delete Messages）

永久删除特定消息：

from langchain.messages import RemoveMessage

def delete_messages(state):
    messages = state["messages"]
    if len(messages) > 2:
        # 删除最早的两条消息
        return {"messages": [RemoveMessage(id=m.id) for m in messages[:2]]}

（3）摘要消息（Summarize Messages）

使用LLM生成摘要，替代旧消息：

from langchain.agents import create_agent
from langchain.agents.middleware import SummarizationMiddleware
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="gpt-4",
    tools=[],
    middleware=[
        SummarizationMiddleware(
            model="gpt-4-mini",
            trigger=("tokens", 4000),  # 超过4000 tokens时触发
            keep=("messages", 20)       # 保留最近20条消息
        )
    ],
    checkpointer=InMemorySaver(),
)

4.2 长期记忆（Long-term Memory）​

长期记忆让Agent能够跨会话、跨线程存储和回忆信息。与短期记忆不同，长期记忆持久化存储用户偏好、知识等信息，可以在不同会话中被检索和使用。

4.2.1 核心概念：Store​

长期记忆基于LangGraph Store实现，数据以JSON文档形式存储，通过namespace（命名空间）和key（键）组织。

4.2.2 基本使用示例​

from langchain.agents import create_agent
from langgraph.store.memory import InMemoryStore

# 创建Store
store = InMemoryStore()

# 创建带长期记忆的Agent
agent = create_agent(
    "claude-sonnet-4-6",
    tools=[],
    store=store,  # 传入Store
)

# 存储数据到Store
user_id = "user_123"
namespace = (user_id, "preferences")
store.put(
    namespace,
    "settings",
    {
        "language": "English",
        "style": "concise",
    }
)

# 从Store检索数据
item = store.get(namespace, "settings")
print(item.value)  # {'language': 'English', 'style': 'concise'}

# 搜索Store（支持向量相似度搜索）
items = store.search(
    namespace,
    filter={"language": "English"},
    query="user preferences"
)

4.2.3 在工具中读写长期记忆​

读取长期记忆：

from langchain.tools import tool, ToolRuntime
from langchain.agents import create_agent
from langgraph.store.memory import InMemoryStore

@tool
def get_user_info(runtime: ToolRuntime) -> str:
    """查询用户信息"""
    assert runtime.store is not None
    user_info = runtime.store.get(("users",), runtime.context.user_id)
    return str(user_info.value) if user_info else "Unknown user"

store = InMemoryStore()
store.put(("users",), "user_123", {"name": "John Smith", "language": "English"})

agent = create_agent(
    model="claude-sonnet-4-6",
    tools=[get_user_info],
    store=store,
)

写入长期记忆：

from langchain.tools import tool, ToolRuntime
from typing_extensions import TypedDict

class UserInfo(TypedDict):
    name: str

@tool
def save_user_info(user_info: UserInfo, runtime: ToolRuntime) -> str:
    """保存用户信息"""
    assert runtime.store is not None
    runtime.store.put(("users",), runtime.context.user_id, dict(user_info))
    return "Successfully saved user info."

4.2.4 生产环境配置​

from langgraph.store.postgres import PostgresStore
from langgraph.store.base import IndexConfig

# 定义嵌入函数（用于向量搜索）
def embed(texts):
    # 使用实际的嵌入模型
    return embedding_model.embed_documents(texts)

DB_URI = "postgresql://postgres:postgres@localhost:5442/postgres?sslmode=disable"

with PostgresStore.from_conn_string(
    DB_URI,
    index=IndexConfig(embed=embed, dims=1536),  # 配置向量索引
) as store:
    store.setup()
    agent = create_agent(
        "claude-sonnet-4-6",
        tools=[save_user_info, get_user_info],
        store=store,
    )

4.3 短期记忆 vs 长期记忆对比​

特性短期记忆长期记忆
作用域单个会话线程跨所有会话
持久化CheckpointerStore
数据格式消息列表JSON文档
检索方式按thread_id加载按namespace/key搜索
典型场景多轮对话上下文用户偏好、知识库
存储后端Memory/PostgreSQL/SQLiteMemory/PostgreSQL

4.4 组合使用示例​

在实际应用中，短期记忆和长期记忆通常组合使用：

from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.store.memory import InMemoryStore

# 创建Agent，同时配置短期和长期记忆
agent = create_agent(
    model="claude-sonnet-4-6",
    tools=[get_user_info, save_user_info],
    checkpointer=InMemorySaver(),  # 短期记忆
    store=InMemoryStore(),          # 长期记忆
)

# 调用时指定thread_id（短期记忆）和context（长期记忆上下文）
result = agent.invoke(
    {"messages": [{"role": "user", "content": "记住我的名字是Bob"}]},
    config={"configurable": {"thread_id": "chat_001"}},
    context=Context(user_id="user_123"),
)

5. 总结​

本文从LLM记忆功能的实现原理出发，介绍了六种常见的记忆模式：

  1. 缓冲记忆：简单直接，适合短对话
  2. 缓冲窗口记忆：固定窗口，token可控
  3. 令牌缓冲记忆：精细token控制
  4. 摘要总结记忆：长对话压缩，适合摘要场景
  5. 摘要缓冲混合记忆：平衡效率与信息保留，推荐使用
  6. 向量存储库记忆：超长对话，语义检索

在LangChain的实现演进上：

版本记忆方案特点
0.3xlangchain_classic.memory手动管理，即将废弃
1.2Checkpointer + Store官方推荐，自动管理

LangChain 1.2版本推荐的记忆架构：

  • 短期记忆：使用Checkpointer实现线程级别的对话历史持久化
  • 长期记忆：使用Store实现跨会话的结构化数据存储

这种设计将记忆的职责清晰分离：短期记忆处理会话上下文，长期记忆处理用户偏好和知识积累，两者可以独立配置、组合使用，为构建复杂AI Agent提供了灵活且强大的记忆能力。

LangChain：Runnable 组件深入

2025年12月28日 · 阅读需 15 分钟
GuguLH
Back-end engineers

前言​

Runnable 组件是 LangChain LCEL 表达式的核心构建块。除了基本的链式调用外，LangChain 还提供了丰富的高级特性，包括参数绑定、运行时配置、组件切换、重试回退机制以及生命周期监听等。本文基于 LangChain 1.2，深入介绍这些高级特性及其应用场景。

1. Runnable 高级特性概述​

1.1 为什么需要这些高级特性？​

在实际生产环境中，我们经常面临以下需求：

场景问题解决方案
参数预设多处调用需要相同参数bind()
运行时配置不同场景需要不同参数configurable_fields()
组件切换A/B 测试、多模型选择configurable_alternatives()
临时故障网络波动、API 限流with_retry()
服务降级主模型不可用with_fallbacks()
可观测性执行监控、日志记录with_listeners()

1.2 方法分类​

Runnable 高级特性
├── 参数配置类
│   ├── bind()                    # 静态绑定参数
│   ├── configurable_fields()     # 运行时配置参数
│   └── configurable_alternatives() # 运行时切换组件
├── 容错机制类
│   ├── with_retry()              # 自动重试
│   └── with_fallbacks()          # 组件回退
└── 可观测性类
    └── with_listeners()          # 生命周期监听

2. bind：动态添加默认调用参数​

2.1 bind 基本用法​

bind() 方法允许我们为 Runnable 预设默认参数，这些参数会在调用时自动传递给底层组件。

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

prompt = ChatPromptTemplate.from_messages([
    ("human", "{query}")
])

llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

# 使用 bind 预设 stop 参数和切换模型
chain = prompt | llm.bind(stop="World").bind(model="minimax-m2.5") | StrOutputParser()

content = chain.invoke({"query": "你是什么模型?"})
print(content)

bind 的核心特点：

  1. 链式调用：可以连续调用多次 bind()
  2. 参数覆盖：后绑定的参数会覆盖先绑定的参数
  3. 延迟生效：绑定的参数在 invoke() 调用时才生效

2.2 bind 解决 RunnableLambda 多参数场景​

当使用 RunnableLambda 包装多参数函数时，bind() 可以预设部分参数，实现类似函数柯里化的效果：

# -*- coding: utf-8 -*-
import random

from langchain_core.runnables import RunnableLambda


def get_weather(location: str, unit: str) -> str:
    """根据传入的位置+温度单位获取对应的天气信息"""
    print("location: ", location)
    print("unit: ", unit)
    return f"{location}天气为{random.randint(24, 40)}{unit}"


# 使用 bind 预设 unit 参数
get_weather_runnable = RunnableLambda(get_weather).bind(unit="摄氏度")

# 调用时只需传入 location
resp = get_weather_runnable.invoke("西安")
print(resp)

输出示例：

location:  西安
unit:  摄氏度
西安天气为28摄氏度

2.3 bind 应用场景​

场景示例
固定模型参数llm.bind(temperature=0)
设置停止词llm.bind(stop=["\n"])
绑定 API 配置llm.bind(api_key="xxx")
预设业务参数RunnableLambda(func).bind(unit="摄氏度")

3. configurable_fields：运行时配置链内部参数​

3.1 基本用法​

configurable_fields() 允许我们在运行时动态配置 Runnable 的参数，比 bind() 更灵活。bind() 是静态绑定，而 configurable_fields() 支持每次调用时传入不同的值。

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import ConfigurableField
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

prompt = ChatPromptTemplate.from_template("请生成一个小于{x}的随机整数")

llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
).configurable_fields(
    temperature=ConfigurableField(
        id="llm_temperature",
        name="大语言模型的温度",
        description="温度越低,大语言模型生成的内容越确定;温度越高,生成的内容越随机"
    )
)

chain = prompt | llm | StrOutputParser()

# 默认温度调用
content = chain.invoke({"x": 1000})
print(content)

print("---------------------------")

# 通过 with_config 设置温度为 0
content = chain.with_config(configurable={"llm_temperature": 0}).invoke({"x": 1000})
print(content)

3.2 两种传参方式​

# 方式一：通过 with_config 方法（推荐，链式调用更优雅）
chain.with_config(configurable={"llm_temperature": 0}).invoke({"x": 1000})

# 方式二：通过 invoke 的 config 参数
chain.invoke(
    {"x": 1000},
    config={"configurable": {"llm_temperature": 0}}
)

3.3 动态替换提示词模板​

configurable_fields 不仅可以配置 LLM 参数，还可以动态替换 Prompt 模板：

# -*- coding: utf-8 -*-
from langchain_core.prompts import PromptTemplate
from langchain_core.runnables import ConfigurableField

# 创建提示模板并配置支持动态配置的字段
prompt = PromptTemplate.from_template("请写一篇关于{subject}的冷笑话").configurable_fields(
    template=ConfigurableField(id="prompt_template")
)

# 传递配置更改 prompt_template 并调用生成内容
content = prompt.invoke(
    {"subject": "程序员"},
    config={"configurable": {"prompt_template": "请写一篇关于{subject}的诗歌"}}
).to_string()
print(content)

输出：

请写一篇关于程序员的诗歌

3.4 bind vs configurable_fields​

特性bind()configurable_fields()
绑定时机定义时固定运行时可变
灵活性低高
使用复杂度简单稍复杂
适用场景参数固定不变需要动态调整

4. configurable_alternatives：动态替换运行组件​

4.1 多 LLM 选择示例​

configurable_alternatives() 允许我们在运行时动态切换不同的组件实现，最典型的场景是切换不同的 LLM 模型：

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import ConfigurableField
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 创建提示词模板&定义默认大语言模型
prompt = ChatPromptTemplate.from_template("{query}")
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
).configurable_alternatives(
    ConfigurableField(id="llm"),
    default_key="glm-5",
    minimax=ChatOpenAI(
        base_url=BASE_URL,
        api_key=API_KEY,
        model="minimax-m2.5",
    ),
    kimi=ChatOpenAI(
        base_url=BASE_URL,
        api_key=API_KEY,
        model="kimi-k2.5",
    ),
)

# 构建链应用
chain = prompt | llm | StrOutputParser()

# 调用链并传递配置信息,进行大模型切换
content = chain.invoke(
    {"query": "你好,你是什么模型?"},
    config={"configurable": {"llm": "minimax"}}
)

print(content)

4.2 参数说明​

参数说明
configurableConfigurableField 实例，定义配置项的 id
default_key默认组件的标识名称
**alternatives备选组件，键为名称，值为 Runnable

4.3 执行流程​

chain.invoke({"query": "你好"}, config={"configurable": {"llm": "minimax"}})
                    │
                    ▼
        ┌───────────────────────┐
        │ 检查 config 中的 llm   │
        │ 配置项                 │
        └───────────┬───────────┘
                    │
         ┌──────────┴──────────┐
         │                     │
    "minimax"              其他/default
         │                     │
         ▼                     ▼
   使用 minimax 模型      使用默认模型

4.4 应用场景​

  • A/B 测试：比较不同模型效果
  • 成本优化：根据任务复杂度选择不同模型
  • 区域部署：不同区域使用不同模型服务
  • 故障演练：验证备用模型可用性

5. with_retry：重试机制​

5.1 基本用法​

with_retry() 为 Runnable 添加自动重试能力，当执行失败时会自动重试：

# -*- coding: utf-8 -*-
from langchain_core.runnables import RunnableLambda

counter = -1


def func(x):
    global counter
    counter += 1
    print(f"当前的值: {counter=}")
    return x / counter


# 配置最多重试 2 次
chain = RunnableLambda(func).with_retry(stop_after_attempt=2)

resp = chain.invoke(2)
print(resp)

执行过程：

当前的值: counter=0  # 第一次执行，除以 0 报错
当前的值: counter=1  # 第一次重试，成功
2.0

5.2 重试参数配置​

from langchain_core.runnables import RunnableLambda

chain = RunnableLambda(func).with_retry(
    stop_after_attempt=3,      # 最大重试次数
    wait_exponential_jitter=True,  # 指数退避 + 抖动
    retry_on_exception=lambda e: isinstance(e, ZeroDivisionError),  # 指定重试的异常类型
)
参数说明
stop_after_attempt最大重试次数
wait_exponential_jitter是否使用指数退避策略
retry_on_exception指定哪些异常需要重试

5.3 适用场景​

  • 网络波动导致的临时性故障
  • API 限流后的自动重试
  • 资源暂时不可用

6. with_fallbacks：回退机制​

6.1 基本用法​

with_fallbacks() 允许设置备选 Runnable，当主 Runnable 执行失败时自动切换到备选组件：

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 构建prompt与LLM
prompt = ChatPromptTemplate.from_template("{query}")

# 主模型配置回退模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model="glm-4.7",  # 可能不存在的模型
).with_fallbacks(
    [
        ChatOpenAI(
            base_url=BASE_URL,
            api_key=API_KEY,
            model=DEFAULT_MODEL,  # 回退模型
        )
    ]
)

# 构建链应用
chain = prompt | llm | StrOutputParser()

# 调用链并输出结果
content = chain.invoke({"query": "你好,你是?"})
print(content)

6.2 执行流程​

chain.invoke({"query": "你好,你是?"})
         │
         ▼
   ┌─────────────┐
   │ 主模型执行   │
   │ glm-4.7     │
   └──────┬──────┘
          │
    ┌─────┴─────┐
    │           │
  成功        失败
    │           │
    ▼           ▼
  返回结果   切换到回退模型
              │
              ▼
         ┌─────────────┐
         │ 回退模型执行 │
         │ DEFAULT_MODEL│
         └──────┬──────┘
                │
                ▼
             返回结果

6.3 重试与回退的区别​

特性with_retrywith_fallbacks
目的临时性错误恢复组件级别的容错
执行内容同一个 Runnable不同的 Runnable
适用场景网络波动、限流模型不可用、服务降级
成本低（同一服务）可能不同（不同服务）

7. with_listeners：生命周期监听器​

7.1 基本用法​

with_listeners() 允许我们为 Runnable 添加生命周期监听函数，在执行的不同阶段执行自定义逻辑：

# -*- coding: utf-8 -*-
import time

from langchain_core.runnables import RunnableLambda
from langchain_core.tracers.schemas import Run
from langchain_core.runnables import RunnableConfig


def on_start(run_obj: Run, config: RunnableConfig) -> None:
    print("on_start")
    print("run_obj: ", run_obj)
    print("config: ", config)
    print("---------------------------")


def on_end(run_obj: Run, config: RunnableConfig) -> None:
    print("on_end")
    print("run_obj: ", run_obj)
    print("config: ", config)
    print("---------------------------")


def on_error(run_obj: Run, config: RunnableConfig) -> None:
    print("on_error")
    print("run_obj: ", run_obj)
    print("config: ", config)
    print("---------------------------")


# 创建 RunnableLambda 与链
runnable = RunnableLambda(lambda x: time.sleep(x)).with_listeners(
    on_start=on_start,
    on_end=on_end,
    on_error=on_error,
)
chain = runnable

# 调用并执行链
chain.invoke(2, config={"configurable": {"name": "GuguLH"}})

7.2 三种核心钩子​

钩子函数触发时机典型用途
on_startRunnable 开始执行前日志记录、初始化资源
on_endRunnable 执行完成后结果处理、资源清理
on_errorRunnable 执行出错时错误处理、告警通知

7.3 钩子函数参数说明​

参数类型说明
run_objRun运行对象，包含执行信息
configRunnableConfig运行配置

Run 对象的主要属性：

run_obj.id           # 运行 ID
run_obj.name         # 运行名称
run_obj.inputs       # 输入数据
run_obj.outputs      # 输出数据（on_end 时可用）
run_obj.error        # 错误信息（on_error 时可用）
run_obj.start_time   # 开始时间
run_obj.end_time     # 结束时间

7.4 实际应用：执行耗时监控​

import time
from langchain_core.runnables import RunnableLambda
from langchain_core.tracers.schemas import Run
from langchain_core.runnables import RunnableConfig


class TimingListener:
    """计时监听器"""

    def __init__(self):
        self.start_time = None

    def on_start(self, run_obj: Run, config: RunnableConfig) -> None:
        self.start_time = time.time()
        print(f"[开始] 执行任务: {run_obj.name}")

    def on_end(self, run_obj: Run, config: RunnableConfig) -> None:
        duration = time.time() - self.start_time
        print(f"[完成] 执行耗时: {duration:.3f}秒")
        print(f"[结果] 输出: {run_obj.outputs}")

    def on_error(self, run_obj: Run, config: RunnableConfig) -> None:
        duration = time.time() - self.start_time
        print(f"[错误] 执行耗时: {duration:.3f}秒")
        print(f"[错误信息] {run_obj.error}")


# 使用监听器
listener = TimingListener()
runnable = RunnableLambda(lambda x: x * 2).with_listeners(
    on_start=listener.on_start,
    on_end=listener.on_end,
    on_error=listener.on_error,
)

result = runnable.invoke(5)

8. 完整示例：生产级链配置​

结合以上技巧，构建一个生产级的链：

import os
import time
from typing import Any

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import ConfigurableField, RunnableConfig
from langchain_core.tracers.schemas import Run
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()


class ProductionListener:
    """生产级监听器"""

    def __init__(self):
        self.start_time = None

    def on_start(self, run_obj: Run, config: RunnableConfig) -> None:
        self.start_time = time.time()
        print(f"[{time.strftime('%Y-%m-%d %H:%M:%S')}] 开始执行")

    def on_end(self, run_obj: Run, config: RunnableConfig) -> None:
        duration = time.time() - self.start_time
        print(f"[{time.strftime('%Y-%m-%d %H:%M:%S')}] 执行完成，耗时: {duration:.3f}秒")

    def on_error(self, run_obj: Run, config: RunnableConfig) -> None:
        print(f"[错误] {run_obj.error}")


# 构建组件
prompt = ChatPromptTemplate.from_template("{query}")

listener = ProductionListener()

# 主模型 + 回退模型
llm = ChatOpenAI(
    base_url=os.getenv("BASE_URL"),
    api_key=os.getenv("API_KEY"),
    model="glm-4-flash",
).configurable_fields(
    temperature=ConfigurableField(id="temperature")
).with_fallbacks([
    ChatOpenAI(
        base_url=os.getenv("BASE_URL"),
        api_key=os.getenv("API_KEY"),
        model=os.getenv("DEFAULT_MODEL"),
    )
]).with_retry(stop_after_attempt=2).with_listeners(
    on_start=listener.on_start,
    on_end=listener.on_end,
    on_error=listener.on_error,
)

# 构建链
chain = prompt | llm | StrOutputParser()

# 调用链
result = chain.invoke(
    {"query": "请用一句话介绍 Python"},
    config={"configurable": {"temperature": 0.5}}
)
print(result)

9. 总结​

9.1 方法速查表​

方法功能典型场景
bind()预设参数固定模型参数、API 配置
configurable_fields()运行时配置参数动态调整温度、切换模板
configurable_alternatives()运行时切换组件多模型选择、A/B 测试
with_retry()自动重试处理临时性错误
with_fallbacks()组件回退服务降级、容错
with_listeners()生命周期监听日志记录、性能监控

9.2 最佳实践​

  1. 生产环境：组合使用 with_retry + with_fallbacks 提高可靠性
  2. 调试阶段：使用 with_listeners 记录详细执行日志
  3. 灵活配置：使用 configurable_fields 和 configurable_alternatives 实现运行时动态调整
  4. 参数预设：使用 bind 简化重复参数传递

9.3 组合使用建议​

推荐的链配置顺序：

llm.configurable_fields(...)   # 1. 运行时参数配置
   .configurable_alternatives(...) # 2. 组件切换（可选）
   .with_fallbacks(...)        # 3. 回退机制
   .with_retry(...)            # 4. 重试机制
   .with_listeners(...)        # 5. 生命周期监听

掌握这些高级特性，可以构建出更加健壮、灵活、可观测的 LangChain 应用。

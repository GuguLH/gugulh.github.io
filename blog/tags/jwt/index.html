JWT 解析：原理、实践与常见问题

2024年4月2日 · 阅读需 9 分钟
GuguLH
Back-end engineers

前言​

JSON Web Token（JWT）已成为现代应用程序中身份认证的通用标准。它以其无状态（Stateless）特性，完美契合了 RESTful API 和微服务架构。本文旨在深入浅出地讲解 JWT 的核心原理、实际应用，并剖析一些开发者在实践中常遇到的问题。

1. 什么是 JWT？​

JWT 的本质是一个遵循特定规范的、紧凑且自包含的 JSON 对象。它被用于在各方之间安全地传输信息。一个完整的 JWT 由三部分组成，用点（.）分隔：

  • Header（头部）：包含令牌的类型（typ）和使用的签名算法（alg）。例如：

    {
      "alg": "HS256",
      "typ": "JWT"
    }

  • Payload（载荷）：包含声明（claims），即一些实体（通常是用户）和附加元数据。声明分为三类：

    • 注册声明（Registered Claims）：预定义的标准字段，如 iss（签发者）、exp（过期时间）、sub（主题）等。
    • 公共声明（Public Claims）：开发者自定义的声明，用于传递公共信息，应避免命名冲突。
    • 私有声明（Private Claims）：用于在特定应用中使用的自定义声明，不建议用于敏感数据。

  • Signature（签名）：由 Base64URL 编码后的 Header、Payload，以及一个密钥（Secret）通过指定的算法（如 HMAC-SHA256）计算得到。它的作用是验证 JWT 是否被篡改。

2. JWT 的工作流程​

JWT 的核心工作流程非常简洁：

  1. 认证：用户使用用户名和密码登录。
  2. 生成令牌：服务器验证凭据后，创建一个包含用户 ID 和其他信息的 JWT，并用密钥对其进行签名。
  3. 返回令牌：服务器将这个 JWT 返回给客户端。
  4. 携带令牌：客户端在后续的每次请求中，将 JWT 放在 HTTP Header 的 Authorization 字段中（通常格式为 Bearer <token>）。
  5. 验证令牌：服务器收到请求后，用相同的密钥验证 JWT 的签名，并解析 Payload 获取用户信息，完成身份认证。

3. 为什么选择 JWT？​

  • 无状态（Stateless）：服务器无需在内存中保存会话信息，减轻了服务器压力，尤其适用于分布式和微服务架构。

  • 跨域兼容：JWT 可以通过 Header 传递，天然支持跨域认证。

  • 自包含：它包含所需的所有信息，服务器无需频繁查询数据库。

4. 实践中的常见问题与最佳实践​

问题一：令牌过期怎么办？​

解决方案：引入 Refresh Token（刷新令牌） 机制。

  • Access Token（访问令牌）：生命周期短（如 15 分钟），用于访问受保护的资源。

  • Refresh Token：生命周期长（如 7 天），用于在 Access Token 过期后，向认证服务器请求新的 Access Token。

这样既保证了安全性，又提升了用户体验。

问题二：如何处理令牌撤销？​

解决方案：

  • 黑名单机制：将已签发但需要提前失效的 JWT 存储在 Redis 或其他缓存中。每次验证时，先检查黑名单。

  • 签发版本号：在 Payload 中加入用户版本号（如 version: 1）。当用户密码修改、权限变更等需要强制所有旧令牌失效时，更新版本号。服务器验证时，如果令牌中的版本号与数据库不一致，则拒绝。

5. 安全性考量与最佳实践​

JWT 虽然强大，但如果使用不当也会引入安全风险。以下几点至关重要：

  1. 保护你的密钥：签名密钥是 JWT 安全的核心。它必须被妥善保管，不能暴露在客户端代码中。建议将密钥存储在环境变量中，或使用专门的密钥管理服务（如 AWS KMS、Vault）。
  2. Payload 并非加密：切记，JWT 的 Payload 只是经过 Base64URL 编码，任何人都可以轻松解码。因此，绝对不要在 Payload 中存放任何敏感信息，如密码、银行卡号等。
  3. 避免使用弱算法：确保你的签名算法是安全的，例如 HMAC-SHA256、RSA-SHA256。要特别注意，避免使用"none"算法，因为这会绕过签名验证，让攻击者可以伪造任意的 Payload。

6. 实践代码示例 (Go)​

我们将使用 github.com/golang-jwt/jwt/v5 这个流行的 Go 库来处理 JWT。

  1. 安装库

    Bash

    go get github.com/golang-jwt/jwt/v5

  2. 生成令牌

    package main

    import (
        "fmt"
        "time"

        "github.com/golang-jwt/jwt/v5"
    )

    // 定义一个自定义的声明结构体
    type UserClaims struct {
        UserID   string `json:"user_id"`
        Username string `json:"username"`
        jwt.RegisteredClaims
    }

    func main() {
        // 建议从环境变量中读取密钥
        var SECRET_KEY = []byte("your_super_secret_key")

        // 创建一个自定义的声明实例
        claims := UserClaims{
            UserID:   "12345",
            Username: "john.doe",
            RegisteredClaims: jwt.RegisteredClaims{
                ExpiresAt: jwt.NewNumericDate(time.Now().Add(time.Hour * 1)), // 令牌过期时间为 1 小时
                IssuedAt:  jwt.NewNumericDate(time.Now()),                    // 令牌签发时间
                Issuer:    "your-app-name",                                   // 签发者
            },
        }

        // 使用 HS256 算法创建令牌
        token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)

        // 使用密钥进行签名
        signedToken, err := token.SignedString(SECRET_KEY)
        if err != nil {
            fmt.Println("Error signing token:", err)
            return
        }

        fmt.Println("Generated token:", signedToken)
    }

  3. 验证令牌

    package main

    import (
        "fmt"
        "github.com/golang-jwt/jwt/v5"
    )

    // 定义一个与生成令牌时相同的自定义声明结构体
    type UserClaims struct {
        UserID   string `json:"user_id"`
        Username string `json:"username"`
        jwt.RegisteredClaims
    }

    func main() {
        var SECRET_KEY = []byte("your_super_secret_key")

        // 假设这是从客户端接收到的令牌
        tokenFromClient := "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoiMTIzNDUiLCJ1c2VybmFtZSI6ImpvaG4uZG9lIiwiZXhwIjoxNzMxNDk0NTcwLCJpYXQiOjE2ODc2NzQwMDAsImlzcyI6InlvdXItYXBwLW5hbWUifQ.H6S-qg4-L-y_s-F6D8b-n6k-g9d-x_6o_m_LgX6o_Xw"

        token, err := jwt.ParseWithClaims(tokenFromClient, &UserClaims{}, func(token *jwt.Token) (interface{}, error) {
            // 验证签名算法是否为 HS256
            if _, ok := token.Method.(*jwt.SigningMethodHMAC); !ok {
                return nil, fmt.Errorf("unexpected signing method: %v", token.Header["alg"])
            }
            return SECRET_KEY, nil
        })

        // 检查解析过程中是否出错
        if err != nil {
            fmt.Println("Token verification failed:", err)
            return
        }

        // 如果令牌有效，获取声明信息
        if claims, ok := token.Claims.(*UserClaims); ok && token.Valid {
            fmt.Println("Token is valid. Decoded payload:")
            fmt.Printf("User ID: %s, Username: %s\n", claims.UserID, claims.Username)
        } else {
            fmt.Println("Token is invalid.")
        }
    }

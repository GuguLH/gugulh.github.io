LangChain：Callback 回调功能及使用技巧

2025年12月27日 · 阅读需 11 分钟
GuguLH
Back-end engineers

前言​

Callback（回调）是 LangChain 中用于监控、调试和记录链执行过程的重要机制。本文基于 LangChain 1.2，介绍 Callback 模块的核心组件和使用技巧。

1. LangChain 中 Callback 功能介绍​

1.1 什么是 Callback？​

在 LangChain 中，Callback 机制允许我们在链执行的各个阶段插入自定义逻辑，例如：

  • 日志记录：记录每个步骤的输入输出
  • 性能监控：统计各阶段耗时
  • 调试排查：追踪执行流程，定位问题
  • 数据采集：收集 token 使用量等指标

1.2 Callback 模块的核心组件​

LangChain 的 Callback 模块主要包含两个核心组件：

组件说明
BaseCallbackHandler回调处理器，定义各种钩子函数
BaseCallbackManager回调管理器，管理多个 Handler

CallbackHandler vs CallbackManager：

┌─────────────────────────────────────────────────┐
│              Runnable (Chain/LLM/Tool)          │
└─────────────────────┬───────────────────────────┘
                      │ 触发事件
                      ▼
┌─────────────────────────────────────────────────┐
│           CallbackManager                        │
│  ┌─────────────┐  ┌─────────────┐               │
│  │ Handler 1   │  │ Handler 2   │  ...          │
│  └─────────────┘  └─────────────┘               │
└─────────────────────────────────────────────────┘

重点：CallbackHandler 是我们最常接触的组件，通过继承 BaseCallbackHandler 并重写钩子函数来实现自定义逻辑。

1.3 CallbackHandler 中的钩子函数​

BaseCallbackHandler 定义了多个 Mixin 类，每个 Mixin 对应一类组件的回调：

Mixin 类钩子函数触发时机
LLMManagerMixinon_llm_startLLM 开始执行前
on_llm_endLLM 执行完成后
on_llm_errorLLM 执行出错时
on_llm_new_token流式输出每个 token 时
ChainManagerMixinon_chain_startChain 开始执行前
on_chain_endChain 执行完成后
on_chain_errorChain 执行出错时
ToolManagerMixinon_tool_startTool 开始执行前
on_tool_endTool 执行完成后
on_tool_errorTool 执行出错时
RetrieverManagerMixinon_retriever_startRetriever 开始执行前
on_retriever_endRetriever 执行完成后
on_retriever_errorRetriever 执行出错时

注意：对于 Chat 模型，使用 on_chat_model_start 而非 on_llm_start。

钩子函数的参数​

每个钩子函数都有特定的参数，例如：

def on_chat_model_start(
    self,
    serialized: dict[str, Any],    # 序列化的模型信息
    messages: list[list[BaseMessage]],  # 输入消息
    *,
    run_id: UUID,                  # 当前运行的唯一标识
    parent_run_id: UUID | None,    # 父运行的 ID（嵌套调用时）
    tags: list[str] | None,        # 标签
    metadata: dict[str, Any] | None,  # 元数据
    **kwargs: Any,
) -> Any:
    pass

def on_llm_end(
    self,
    response: LLMResult,           # LLM 的响应结果
    *,
    run_id: UUID,
    parent_run_id: UUID | None = None,
    tags: list[str] | None = None,
    **kwargs: Any,
) -> Any:
    pass

钩子函数触发时序图​

chain.invoke(input)
    │
    ├── on_chain_start
    │
    ├── on_chat_model_start (或 on_llm_start)
    │       │
    │       ├── on_llm_new_token (流式输出时，每个 token)
    │       │
    │       └── on_llm_end / on_llm_error
    │
    └── on_chain_end / on_chain_error

2. 在 LangChain 中使用回调​

2.1 使用 CallbackHandler 的几种方式​

LangChain 提供了多种方式来使用回调：

方式一：通过 config 参数传递（推荐）​

from langchain_core.callbacks import StdOutCallbackHandler

chain = prompt | llm | parser

# 在调用时传入 callbacks
result = chain.invoke(
    {"query": "你好"},
    config={"callbacks": [StdOutCallbackHandler()]}
)

优点：

  • 灵活性高，每次调用可以使用不同的回调
  • 不影响全局配置
  • 推荐在生产环境中使用

方式二：构造函数中传入（不推荐）​

# 在构造 LLM 时传入 callbacks
llm = ChatOpenAI(
    callbacks=[StdOutCallbackHandler()],
    # ...其他参数
)

缺点：

  • 回调与组件强绑定，不够灵活
  • 难以在运行时动态切换回调

方式三：通过 with_config 方法（推荐）​

from langchain_core.callbacks import StdOutCallbackHandler

# 使用 with_config 方法绑定配置，返回新的 Runnable
chain_with_callbacks = chain.with_config(
    callbacks=[StdOutCallbackHandler()]
)

# 调用时无需再传入 callbacks
result = chain_with_callbacks.invoke({"query": "你好"})

优点：

  • 预先绑定配置，调用时更简洁
  • 返回新的 Runnable，不影响原链
  • 适合需要多次使用相同配置的场景

方式四：通过 RunnableConfig 配置​

from langchain_core.callbacks import StdOutCallbackHandler

# 创建配置
config = {
    "callbacks": [StdOutCallbackHandler()],
    "tags": ["debug", "production"],
    "metadata": {"user_id": "123"}
}

result = chain.invoke({"query": "你好"}, config=config)

优点：

  • 可以同时配置 callbacks、tags、metadata
  • 配置集中管理

2.2 内置的 CallbackHandler​

LangChain 提供了几个内置的 Handler：

Handler说明
StdOutCallbackHandler输出到标准输出（控制台）
FileCallbackHandler输出到文件
StreamingStdOutCallbackHandler流式输出到标准输出

StdOutCallbackHandler 示例​

from langchain_core.callbacks import StdOutCallbackHandler

handler = StdOutCallbackHandler()

result = chain.invoke(
    {"query": "你好"},
    config={"callbacks": [handler]}
)

输出示例：

> Entering new RunnableSequence chain...

> Finished chain.

FileCallbackHandler 示例​

from langchain_core.callbacks import FileCallbackHandler

handler = FileCallbackHandler("log.txt")

result = chain.invoke(
    {"query": "你好"},
    config={"callbacks": [handler]}
)

2.3 推荐与不推荐的方式​

方式推荐度说明
config={"callbacks": [...]}推荐灵活、清晰、易维护
with_config(callbacks=[...])推荐预先绑定，适合多次复用
构造函数传入不推荐绑定过紧，不易测试
全局设置不推荐副作用难以追踪

3. 自定义回调​

3.1 创建自定义 CallbackHandler​

通过继承 BaseCallbackHandler 并重写需要的钩子函数：

# -*- coding: utf-8 -*-
import os
import time
from typing import Any
from uuid import UUID

import dotenv
from langchain_core.messages import BaseMessage
from langchain_core.output_parsers import StrOutputParser
from langchain_core.outputs import LLMResult
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.callbacks import StdOutCallbackHandler, BaseCallbackHandler
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")


class LLMOpsCallbackHandler(BaseCallbackHandler):
    """自定义LLMOps回调处理器"""

    start_at: float = 0

    def on_chat_model_start(
            self,
            serialized: dict[str, Any],
            messages: list[list[BaseMessage]],
            *,
            run_id: UUID,
            parent_run_id: UUID | None = None,
            tags: list[str] | None = None,
            metadata: dict[str, Any] | None = None,
            **kwargs: Any,
    ) -> Any:
        print("聊天模型开始执行...")
        print("serialized: ", serialized)
        print("inputs: ", messages)
        self.start_at = time.time()

    def on_llm_end(
            self,
            response: LLMResult,
            *,
            run_id: UUID,
            parent_run_id: UUID | None = None,
            tags: list[str] | None = None,
            **kwargs: Any,
    ) -> Any:
        end_at: float = time.time()
        print("完整输出: ", response)
        print("程序消耗: ", end_at - self.start_at)


# 1 编排prompt
prompt = ChatPromptTemplate.from_template("{query}")

# 2 创建大语言模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

# 3 构建链
chain = {"query": RunnablePassthrough()} | prompt | llm | StrOutputParser()

# 4 调用链并执行
resp = chain.stream(
    "你好,你是?",
    config={"callbacks": [StdOutCallbackHandler(), LLMOpsCallbackHandler()]}
)

for chunk in resp:
    pass

输出示例：

> Entering new RunnableSequence chain...

聊天模型开始执行...
serialized:  {'id': ['langchain', 'chat_models', 'openai', 'ChatOpenAI'], 'lc': 1, ...}
inputs:  [[HumanMessage(content='你好,你是?')]]
完整输出:  llm_output={'token_usage': {...}, ...}
程序消耗:  1.234秒

> Finished chain.

3.3 实现完整的监控回调​

import time
from typing import Any
from uuid import UUID

from langchain_core.callbacks import BaseCallbackHandler
from langchain_core.messages import BaseMessage
from langchain_core.outputs import LLMResult


class MetricsCallbackHandler(BaseCallbackHandler):
    """性能监控回调处理器"""

    def __init__(self):
        self.start_times: dict[UUID, float] = {}
        self.metrics: dict[str, Any] = {}

    def on_chain_start(
        self,
        serialized: dict[str, Any],
        inputs: dict[str, Any],
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> Any:
        self.start_times[run_id] = time.time()
        self.metrics["chain_input"] = inputs

    def on_chain_end(
        self,
        outputs: dict[str, Any],
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> Any:
        duration = time.time() - self.start_times.get(run_id, 0)
        self.metrics["chain_output"] = outputs
        self.metrics["chain_duration"] = duration
        print(f"Chain 执行耗时: {duration:.3f}秒")

    def on_chat_model_start(
        self,
        serialized: dict[str, Any],
        messages: list[list[BaseMessage]],
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> Any:
        self.start_times[run_id] = time.time()

    def on_llm_end(
        self,
        response: LLMResult,
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> Any:
        duration = time.time() - self.start_times.get(run_id, 0)
        token_usage = response.llm_output.get("token_usage", {})
        self.metrics["llm_duration"] = duration
        self.metrics["token_usage"] = token_usage
        print(f"LLM 执行耗时: {duration:.3f}秒")
        print(f"Token 使用: {token_usage}")

3.4 同时使用多个 Handler​

# 可以同时使用多个 Handler
handlers = [
    StdOutCallbackHandler(),      # 控制台输出
    FileCallbackHandler("log.txt"), # 文件日志
    MetricsCallbackHandler(),      # 性能监控
]

result = chain.invoke(
    {"query": "你好"},
    config={"callbacks": handlers}
)

4. 异步回调​

对于异步应用，可以使用 AsyncCallbackHandler：

from langchain_core.callbacks import AsyncCallbackHandler

class AsyncMetricsCallbackHandler(AsyncCallbackHandler):
    """异步回调处理器"""

    async def on_chat_model_start(
        self,
        serialized: dict[str, Any],
        messages: list[list[BaseMessage]],
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> None:
        print("异步：模型开始执行")

    async def on_llm_end(
        self,
        response: LLMResult,
        *,
        run_id: UUID,
        **kwargs: Any,
    ) -> None:
        print("异步：模型执行完成")

# 异步调用
result = await chain.ainvoke(
    {"query": "你好"},
    config={"callbacks": [AsyncMetricsCallbackHandler()]}
)

5. 完整示例​

import os
import time
from typing import Any
from uuid import UUID

import dotenv
from langchain_core.callbacks import BaseCallbackHandler, StdOutCallbackHandler
from langchain_core.messages import BaseMessage
from langchain_core.output_parsers import StrOutputParser
from langchain_core.outputs import LLMResult
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()


class DebugCallbackHandler(BaseCallbackHandler):
    """调试回调处理器"""

    def __init__(self):
        self.start_time = 0

    def on_chain_start(
        self,
        serialized: dict[str, Any],
        inputs: dict[str, Any],
        **kwargs: Any,
    ) -> Any:
        print(f"[Chain] 开始执行，输入: {inputs}")

    def on_chain_end(
        self,
        outputs: dict[str, Any],
        **kwargs: Any,
    ) -> Any:
        print(f"[Chain] 执行完成")

    def on_chat_model_start(
        self,
        serialized: dict[str, Any],
        messages: list[list[BaseMessage]],
        **kwargs: Any,
    ) -> Any:
        self.start_time = time.time()
        print(f"[LLM] 开始调用模型")

    def on_llm_end(
        self,
        response: LLMResult,
        **kwargs: Any,
    ) -> Any:
        duration = time.time() - self.start_time
        tokens = response.llm_output.get("token_usage", {})
        print(f"[LLM] 调用完成，耗时: {duration:.3f}秒")
        print(f"[LLM] Token使用: prompt={tokens.get('prompt_tokens')}, "
              f"completion={tokens.get('completion_tokens')}")


# 构建链
prompt = ChatPromptTemplate.from_template("{query}")
llm = ChatOpenAI(
    base_url=os.getenv("BASE_URL"),
    api_key=os.getenv("API_KEY"),
    model=os.getenv("DEFAULT_MODEL"),
)
chain = {"query": RunnablePassthrough()} | prompt | llm | StrOutputParser()

# 使用回调
result = chain.invoke(
    "请讲一个程序员冷笑话",
    config={"callbacks": [StdOutCallbackHandler(), DebugCallbackHandler()]}
)

print(f"\n最终结果: {result}")

要点说明
核心组件BaseCallbackHandler（处理器）和 BaseCallbackManager（管理器）

Callback 机制是 LangChain 中调试和监控链执行的重要工具。通过自定义 Handler，可以灵活地实现日志记录、性能监控、错误追踪等功能，为生产环境的可观测性提供支持。

LangChain：LCEL表达式以及Runnable 核心类

2025年12月27日 · 阅读需 12 分钟
GuguLH
Back-end engineers

前言​

LCEL（LangChain Expression Language）是 LangChain 的表达式语言，它让我们可以用简洁的语法构建处理链。本文基于 LangChain 1.2，从手写一个简易 Chain 开始，深入理解 LCEL 的核心原理，并介绍两个常用的 Runnable 核心类。

1. 手写一个 Chain，搞清楚 LCEL 的 invoke 核心原理​

在使用 LCEL 之前，先来看一个最朴素的问题：如何把 Prompt、Model、Parser 串联起来？

1.1 朴素的手写 Chain​

# -*- coding: utf-8 -*-
import os
from typing import List, Any

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 1 构建组件
prompt = ChatPromptTemplate.from_template("{query}")
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)
parser = StrOutputParser()


# 2 定义一个链
class Chain:
    steps: List = []

    def __init__(self, steps: List):
        self.steps = steps

    def invoke(self, input_data: Any) -> Any:
        for step in self.steps:
            input_data = step.invoke(input_data)
            print("步骤: ", step)
            print("输出: ", input_data)
            print("---------------------------")
        return input_data


# 3 编排链
chain = Chain([prompt, llm, parser])

# 4 执行链并获取结果
print(chain.invoke({"query": "你好,你是?"}))

执行流程：

步骤:  ChatPromptTemplate(...)
输出:  messages=[HumanMessage(content='你好,你是?')]
---------------------------
步骤:  ChatOpenAI(...)
输出:  content='你好！我是由OpenAI开发的...' response_metadata={...}
---------------------------
步骤:  StrOutputParser()
输出:  你好！我是由OpenAI开发的...
---------------------------

1.2 invoke 的核心原理​

从这个简易 Chain 可以看出 LCEL 的核心原理：

  1. 每个组件都有 invoke() 方法：接收输入，返回输出
  2. 链式调用：前一个组件的输出作为后一个组件的输入
  3. 顺序执行：按编排顺序依次调用每个步骤
输入数据 → Step1.invoke() → Step2.invoke() → Step3.invoke() → 最终结果

这就是 LCEL 的本质——一个优雅的链式调用封装。

2. LangChain 里的 Runnable 协议​

2.1 为什么可以使用 | 构建链？​

在 LangChain 中，我们可以用管道符 | 来构建链：

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 1 构建组件
prompt = ChatPromptTemplate.from_template("{query}")
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)
parser = StrOutputParser()

# 2 创建链
chain = prompt | llm | parser

# 3 调用链得到结果
print(chain.invoke({"query": "请讲一个关于程序员的冷笑话"}))

为什么 | 能工作？

这是 Python 的魔法方法 __or__ 和 __ror__ 的功劳：

# 等价于
chain = prompt.__or__(llm).__or__(parser)

2.2 Runnable 协议​

LangChain 中所有可组合的组件都实现了 Runnable 协议。查看 langchain_core 源码，可以发现：

from langchain_core.runnables import Runnable

# Prompt、Model、Parser 都继承自 Runnable
isinstance(prompt, Runnable)   # True
isinstance(llm, Runnable)      # True
isinstance(parser, Runnable)   # True

Runnable 协议的核心方法：

方法说明
invoke(input)同步执行，返回单个结果
batch(inputs)批量执行，返回结果列表
stream(input)流式执行，返回迭代器
ainvoke(input)异步执行
pipe(other)与另一个 Runnable 连接（ | 的底层实现）

__or__ 方法的实现：

class Runnable:
    def __or__(self, other):
        """使用 | 连接两个 Runnable"""
        return RunnableSequence(first=self, second=other)

所以 prompt | llm | parser 实际创建了一个 RunnableSequence：

chain = prompt | llm | parser
print(type(chain))  # <class 'langchain_core.runnables.base.RunnableSequence'>

2.3 Runnable 协议的优势​

特性说明
统一接口所有组件都有 invoke/batch/stream 方法
自动转换字典、函数等可以自动转为 Runnable
嵌套组合Runnable 可以嵌套组合成复杂结构
异步支持所有 Runnable 都支持异步调用

3. 两个常用的 Runnable 核心类​

3.1 RunnableParallel：并行执行多个链​

RunnableParallel 允许同时执行多个分支，并将结果合并为一个字典。

基本用法​

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnableParallel
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 1 编排 Prompt
joke_prompt = ChatPromptTemplate.from_template("请讲一个关于{subject}的冷笑话,尽可能简短一些")
poem_prompt = ChatPromptTemplate.from_template("请写一篇关于{subject}的诗")

# 2 创建大语言模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
    extra_body={"reasoning_split": True},
)

# 3 创建输出解析器
parser = StrOutputParser()

# 4 编排链
joke_chain = joke_prompt | llm | parser
poem_chain = poem_prompt | llm | parser

# 5 并行链
map_chain = RunnableParallel(joke=joke_chain, poem=poem_chain)

ret = map_chain.invoke({"subject": "程序员"})
print(ret)

输出示例：

{
    'joke': '为什么程序员总是分不清万圣节和圣诞节？\n因为 Oct 31 = Dec 25',
    'poem': '键盘敲响代码声，\n显示器前夜已深。\nBug修复心欢喜，\n程序运行梦成真。'
}

两种写法​

RunnableParallel 支持两种写法：

# 写法一：关键字参数
map_chain = RunnableParallel(joke=joke_chain, poem=poem_chain)

# 写法二：字典形式
map_chain = RunnableParallel({
    "joke": joke_chain,
    "poem": poem_chain,
})

简写形式​

字典会自动转换为 RunnableParallel：

# 这两种写法等价
chain = RunnableParallel({"joke": joke_chain, "poem": poem_chain})
chain = {"joke": joke_chain, "poem": poem_chain}  # 自动转换

3.2 RunnableParallel 模拟检索场景​

在 RAG（检索增强生成）场景中，需要同时传入检索结果和用户问题：

3.3 RunnablePassthrough：简化 invoke 调用​

RunnablePassthrough 用于将输入原样传递，常用于以下场景：

场景一：保留原始输入​

from langchain_core.runnables import RunnablePassthrough

# RunnablePassthrough() 将输入原样传递
chain = {
    "context": retrieval,           # 调用检索函数
    "query": RunnablePassthrough(), # 原样传递输入
} | prompt | llm | parser

# 调用时直接传字符串
chain.invoke("你好,我是谁?")

场景二：使用 assign 动态添加字段​

import dotenv
from langchain_core.output_parsers import StrOutputParser

# 输出

写法代码适用场景
字典 + lambda{"context": lambda x: retrieval(x["query"])}需要从字典中提取字段
字典 + itemgetter{"query": itemgetter("query")}简单字段提取
RunnablePassthrough{"query": RunnablePassthrough()}原样传递整个输入
RunnablePassthrough.assign.assign(context=lambda x: ...)动态添加字段

结合前面所学，来看一个完整的 RAG 风格链：

import os
from operator import itemgetter

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()


def retrieval(query: str) -> str:
    """模拟检索函数"""
    return "检索到的上下文内容..."


# 1. Prompt 模板
prompt = ChatPromptTemplate.from_template("""
根据以下上下文回答用户问题：

上下文：{context}
问题：{question}
""")

# 2. 模型
llm = ChatOpenAI(
    base_url=os.getenv("BASE_URL"),
    api_key=os.getenv("API_KEY"),
    model=os.getenv("DEFAULT_MODEL"),
)

# 3. 解析器
parser = StrOutputParser()

# 4. 构建链（三种等价写法）

# 写法一：字典形式
chain1 = {
    "context": lambda x: retrieval(x["question"]),
    "question": itemgetter("question"),
} | prompt | llm | parser

# 写法二：RunnablePassthrough.assign
chain2 = (
    RunnablePassthrough.assign(context=lambda x: retrieval(x["question"]))
    | prompt
    | llm
    | parser
)

# 写法三：RunnablePassthrough 原样传递（输入直接是字符串）
chain3 = (
    {"context": retrieval, "question": RunnablePassthrough()}
    | prompt
    | llm
    | parser
)

# 调用
result1 = chain1.invoke({"question": "什么是Python？"})
result2 = chain2.invoke({"question": "什么是Python？"})
result3 = chain3.invoke("什么是Python？")

5. 总结​

组件作用典型用法
RunnableSequence顺序执行prompt | llm | parser
RunnableParallel并行执行{"a": chain1, "b": chain2}
RunnablePassthrough原样传递{"query": RunnablePassthrough()}
RunnablePassthrough.assign动态添加字段.assign(context=lambda x: ...)

LCEL 的设计哲学：

  1. 统一接口：所有组件都实现 Runnable 协议
  2. 声明式语法：用 | 直观地表达数据流向
  3. 自动转换：字典、函数等自动转为 Runnable
  4. 灵活组合：支持嵌套、并行、条件分支等复杂结构

掌握 LCEL 和 Runnable 核心类，是高效使用 LangChain 的基础。

LangChain：OutputParser 组件及使用技巧

2025年12月27日 · 阅读需 13 分钟
GuguLH
Back-end engineers

前言​

OutputParser（输出解析器）是 LangChain 中用于将大语言模型的原始输出转换为结构化数据的组件。本文基于 LangChain 1.2，结合示例代码，聊聊 OutputParser 组件的使用技巧和常见问题解决方案。

1. 为什么需要输出解析器​

大语言模型的输出本质上是文本字符串，但实际应用中我们往往需要：

  • 结构化数据：如 JSON 格式，方便程序处理
  • 特定格式：如列表、日期、数字等
  • 类型安全：确保输出符合预期类型

例如，当我们要构建一个问答系统，希望模型返回结构化的答案：

用户问：请讲一个关于程序员的冷笑话

期望输出（JSON）：
{
    "joke": "为什么程序员总是分不清万圣节和圣诞节？",
    "punchline": "因为 Oct 31 = Dec 25"
}

这就是 OutputParser 的用武之地——它帮助我们：

  1. 指导模型输出格式：通过提示词告诉模型应该输出什么格式
  2. 解析原始输出：将文本转换为结构化数据
  3. 错误处理：当解析失败时提供重试或修复机制

2. OutputParser 输出解析器​

LangChain 提供了多种输出解析器，最常用的是 StrOutputParser 和 JsonOutputParser。

2.1 StrOutputParser：最简单的字符串解析器​

StrOutputParser 是最基础的解析器，它将模型的 AIMessage 对象转换为纯字符串。

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")

# 1. 编排提示模板
prompt = ChatPromptTemplate.from_template("{query}")

# 2. 构建大语言模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

# 3. 构建字符串输出解析器
parser = StrOutputParser()

# 4. 调用大语言模型生成结果并解析
content = parser.invoke(llm.invoke(prompt.invoke({"query": "你好,你是哪个模型?"})))
print(content)

输出示例：

你好！我是由OpenAI开发的GPT模型，很高兴为你服务。有什么我可以帮助你的吗？

使用场景：

  • 只需要模型的文本回复，不需要额外结构化
  • 作为 LCEL（LangChain Expression Language）链的终端组件

LCEL 链式调用：

更优雅的写法是使用管道符 | 构建链式调用：

chain = prompt | llm | StrOutputParser()
result = chain.invoke({"query": "你好,你是哪个模型?"})
print(result)

2.2 JsonOutputParser：结构化 JSON 输出​

JsonOutputParser 用于获取结构化的 JSON 输出，通常结合 Pydantic 模型定义数据结构。

# -*- coding: utf-8 -*-
import os

import dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

dotenv.load_dotenv()

API_KEY = os.getenv("API_KEY")
BASE_URL = os.getenv("BASE_URL")
DEFAULT_MODEL = os.getenv("DEFAULT_MODEL")


# 1. 创建一个 JSON 数据结构，用于告诉大语言模型这个 JSON 长什么样子
class Joke(BaseModel):
    # 冷笑话
    joke: str = Field(description="回答用户的冷笑话")
    # 冷笑话的笑点
    punchline: str = Field(description="这个冷笑话的笑点在哪")


parser = JsonOutputParser(pydantic_object=Joke)

# 2. 构建一个提示模板
prompt = (ChatPromptTemplate.from_template("请根据用户的提问进行回答.\n{format_instructions}\n{query}")
          .partial(format_instructions=parser.get_format_instructions()))

# 3. 构建大语言模型
llm = ChatOpenAI(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

# 4. 传递提示并进行解析
joke = parser.invoke(llm.invoke(prompt.invoke({"query": "请讲一个关于程序员的冷笑话"})))
print(type(joke))
print(joke.get("punchline"))
print(joke)

输出示例：

<class 'dict'>
因为 Oct 31 = Dec 25
{'joke': '为什么程序员总是分不清万圣节和圣诞节？', 'punchline': '因为 Oct 31 = Dec 25'}

关键点解析：

步骤说明
定义 Pydantic 模型使用 BaseModel 和 Field 描述字段及其含义
创建解析器JsonOutputParser(pydantic_object=Joke)
获取格式指令parser.get_format_instructions() 生成提示词模板
解析输出parser.invoke() 将 AIMessage 转为字典

get_format_instructions() 生成的提示词：

STRICT OUTPUT FORMAT:
- Return only the JSON value that conforms to the schema. Do not include any additional text, explanations, headings, or separators.
- Do not wrap the JSON in Markdown or code fences (no ``` or ```json).
- Do not prepend or append any text (e.g., do not write "Here is the JSON:").
- The response must be a single top-level JSON value exactly as required by the schema (object/array/etc.), with no trailing commas or comments.

The output should be formatted as a JSON instance that conforms to the JSON schema below.

As an example, for the schema {"properties": {"foo": {"title": "Foo", "description": "a list of strings", "type": "array", "items": {"type": "string"}}}, "required": ["foo"]} the object {"foo": ["bar", "baz"]} is a well-formatted instance of the schema. The object {"properties": {"foo": ["bar", "baz"]}} is not well-formatted.

Here is the output schema (shown in a code block for readability only — do not include any backticks or Markdown in your output):
```
{"properties": {"joke": {"description": "回答用户的冷笑话", "title": "Joke", "type": "string"}, "punchline": {"description": "这个冷笑话的笑点在哪", "title": "Punchline", "type": "string"}}, "required": ["joke", "punchline"]}
```

这段提示词会被注入到 Prompt 中，指导模型输出正确格式的 JSON。

3. 工具调用 vs 输出解析器​

3.1 如果模型支持工具调用，可以不使用解析器​

现代大语言模型（如 GPT-4、Claude 等）支持函数调用/工具调用功能，可以更可靠地输出结构化数据：

from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

class Joke(BaseModel):
    joke: str = Field(description="回答用户的冷笑话")
    punchline: str = Field(description="这个冷笑话的笑点在哪")

llm = ChatOpenAI(model="gpt-4")

# 使用 with_structured_output 方法，直接绑定结构化输出
structured_llm = llm.with_structured_output(Joke)

# 直接返回 Pydantic 对象，无需解析器
result = structured_llm.invoke("请讲一个关于程序员的冷笑话")
print(result.joke)        # 访问属性
print(result.punchline)   # 访问属性

工具调用的优势：

对比项OutputParser工具调用
原理提示词引导 + 文本解析模型原生支持
可靠性依赖模型遵循指令模型内部保证格式
适用模型所有模型仅支持工具调用的模型
输出类型字典Pydantic 对象

选择建议：

  • 如果模型支持工具调用，优先使用 with_structured_output()
  • 如果模型不支持工具调用，再使用 JsonOutputParser

4. 输出解析失效的解决方案​

OutputParser 有时会失效，常见原因和解决方案如下：

4.1 常见失效原因​

原因表现
模型输出格式不符合预期JSON 缺少字段、类型错误
输出包含额外文本"好的，这是你的 JSON：{...}"
JSON 截断输出太长被截断，JSON 不完整
特殊字符问题引号、换行符处理不当

4.2 解决方案​

方案一：增强提示词

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的助手，必须严格按照JSON格式输出，不要包含任何其他文字。"),
    ("human", "{query}\n\n请直接输出JSON，不要有任何前缀或后缀说明。")
])

方案二：手动提取 JSON

import json
import re

def extract_json(text: str) -> dict:
    """从文本中提取JSON"""
    # 尝试直接解析
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass

    # 使用正则提取 JSON 块
    json_pattern = r'\{[^{}]*\}'
    matches = re.findall(json_pattern, text, re.DOTALL)
    for match in matches:
        try:
            return json.loads(match)
        except json.JSONDecodeError:
            continue

    raise ValueError("无法从输出中提取有效的JSON")

5. 小参数模型的最佳实践​

针对小参数模型（如 7B、13B 等小模型），且不支持工具调用的情况，选择合适的输出解析器至关重要。

5.1 推荐方案：StrOutputParser + 正则提取​

小参数模型对复杂 JSON 格式的遵循能力较弱，建议：

from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
import re

# 使用更简单的格式要求
prompt = ChatPromptTemplate.from_template("""
请回答用户的问题，并按以下格式输出：

笑话：[你的笑话内容]
笑点：[笑点解释]

用户问题：{query}
""")

chain = prompt | llm | StrOutputParser()
result = chain.invoke({"query": "讲一个程序员冷笑话"})

# 使用正则提取
joke_match = re.search(r"笑话[：:]\s*(.+)", result)
punchline_match = re.search(r"笑点[：:]\s*(.+)", result)

joke_data = {
    "joke": joke_match.group(1).strip() if joke_match else None,
    "punchline": punchline_match.group(1).strip() if punchline_match else None
}
print(joke_data)

5.2 为什么这样更稳定？​

解析方式小模型适用性原因
JsonOutputParser较差JSON 格式要求严格，小模型容易出错
StrOutputParser + 正则较好格式宽松，容错性强
工具调用不适用小模型通常不支持

5.3 提高稳定性的技巧​

# 1. 使用更简单的输出格式
prompt = ChatPromptTemplate.from_template("""
回答格式要求：
- 第一行：笑话内容
- 第二行：笑点解释

用户问题：{query}
""")

# 2. 多次重试
def parse_with_retry(chain, query, max_retries=3):
    for i in range(max_retries):
        try:
            result = chain.invoke({"query": query})
            # 验证结果...
            return result
        except Exception as e:
            if i == max_retries - 1:
                raise
            continue

# 3. Few-shot 示例
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个专业的助手。"),
    ("human", "讲一个程序员冷笑话"),
    ("assistant", "笑话：为什么程序员总是分不清万圣节和圣诞节？\n笑点：因为 Oct 31 = Dec 25"),
    ("human", "{query}")
])

6. 完整示例：LCEL 链式调用​

结合前面的知识，来看一个完整的链式调用示例：

import os
import dotenv
from langchain_core.output_parsers import StrOutputParser, JsonOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

dotenv.load_dotenv()

# 定义数据结构
class Joke(BaseModel):
    joke: str = Field(description="回答用户的冷笑话")
    punchline: str = Field(description="这个冷笑话的笑点在哪")

# 创建组件
llm = ChatOpenAI(
    base_url=os.getenv("BASE_URL"),
    api_key=os.getenv("API_KEY"),
    model=os.getenv("DEFAULT_MODEL"),
)
json_parser = JsonOutputParser(pydantic_object=Joke)

# 方式一：字符串输出链
str_chain = ChatPromptTemplate.from_template("{query}") | llm | StrOutputParser()

# 方式二：JSON 输出链
json_chain = (
    ChatPromptTemplate.from_template(
        "请根据用户的提问进行回答.\n{format_instructions}\n{query}"
    ).partial(format_instructions=json_parser.get_format_instructions())
    | llm
    | json_parser
)

# 调用
print("=== 字符串输出 ===")
print(str_chain.invoke({"query": "你好，你是谁？"}))

print("\n=== JSON 输出 ===")
result = json_chain.invoke({"query": "讲一个程序员冷笑话"})
print(f"笑话：{result['joke']}")
print(f"笑点：{result['punchline']}")

7. 总结​

场景推荐方案说明
简单文本输出StrOutputParser最基础的解析器
结构化输出（大模型）with_structured_output()模型原生支持，最可靠
结构化输出（普通模型）JsonOutputParser提示词引导 + 解析
结构化输出（小模型）StrOutputParser + 正则格式宽松，容错性强

LangChain 1.2 的 OutputParser 组件设计简洁，核心思想是：通过提示词指导模型输出格式，再通过解析器将文本转换为结构化数据。掌握不同场景下的选择策略，能够有效提升应用的稳定性和可靠性。

GuguLH
Back-end engineers

前言​

Model 组件是 LangChain 的核心，它封装了对大语言模型的调用。本文基于 LangChain 1.2，结合官方示例代码，聊聊 Model 组件的几种常见用法。

1. 先跑起来：最基础的用法​

直接看代码：

import os
from datetime import datetime

import dotenv
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

# 创建模型实例
llm = ChatOpenAI(
    api_key=os.getenv("API_KEY"),
    base_url=os.getenv("BASE_URL"),
    model=os.getenv("DEFAULT_MODEL"),
)

# 编排 Prompt
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是OpenAI开发的聊天机器人,请回答用户的问题,现在的时间是{now}"),
        ("human", "{query}")
    ]
).partial(now=datetime.now)

# 调用模型
ai_message = llm.invoke(prompt.invoke({
    "query": "现在是几点?我现在应该做什么?",
}))
print(ai_message.content)

输出示例：

现在是下午3点45分。根据您目前的时间，我建议您可以：

1. 喝杯咖啡休息一下
2. 处理一些需要集中精力的工作
3. 站起来活动活动

有什么我可以帮助您的吗？

几点说明：

  • ChatOpenAI 是 OpenAI 模型的封装，来自 langchain_openai 包
  • invoke() 是核心方法，接收 PromptValue 对象，返回 AIMessage
  • ai_message.content 是模型的文本回复
  • ai_message.response_metadata 包含响应元数据，如 token 使用量、模型名称等

1.1 response_metadata 详解​

print(ai_message.response_metadata)

输出示例：

{
    'token_usage': {'completion_tokens': 86, 'prompt_tokens': 58, 'total_tokens': 144},
    'model_name': 'gpt-3.5-turbo',
    'system_fingerprint': 'fp_1234567890',
    'finish_reason': 'stop',
    'logprobs': None
}

这个元数据对于监控模型使用、调试应用非常有用。

2. 批处理：一次处理多个请求​

如果需要同时处理多个请求，用 batch() 方法可以显著提升效率：

# 准备多个请求
requests = [
    prompt.invoke({"query": "你好,你是?"}),
    prompt.invoke({"query": "请讲一个关于程序员的冷笑话"}),
    prompt.invoke({"query": "用一句话介绍Python")],
]

# 批量调用
ai_messages = llm.batch(requests)

for ai_message in ai_messages:
    print(ai_message.content)
    print("---------------------------")

输出：

你好！我是由OpenAI开发的AI助手，可以帮助您回答各种问题。有什么我可以帮助您的吗？
---------------------------
为什么程序员总是分不清万圣节和圣诞节？

因为 Oct 31 = Dec 25

（Octal 31 = Decimal 25）
---------------------------
Python是一门简洁优雅的高级编程语言，以可读性和易用性著称。
---------------------------

优势对比：

方式耗时适用场景
循环调用 invoke()O(n) × 单次耗时请求少、无并发需求
batch() 批量调用O(1) 或 O(批次)多个独立请求、需要批量处理

batch() 的原理是将多个请求打包发送给模型服务商，由服务商内部并行处理，返回结果列表。

3. 流式输出：实时看到生成过程​

对于需要实时反馈的场景，比如聊天机器人，流式输出很有用：

response = llm.stream(prompt.invoke({
    "query": "你能让我极易理解的方式,讲清楚大模型原理吗?"
}))

for chunk in response:
    print(chunk.content, end="", flush=True)

输出是逐字/逐词显示的，用户可以看到模型"思考"的过程，而不是等待完整回答。

流式输出的特点：

  • 返回的是生成器，每次迭代得到一个 AIMessageChunk
  • chunk.content 是本轮的增量内容
  • 适合长文本生成场景，用户可以提前看到部分结果
  • 常用于聊天界面、终端输出等需要实时反馈的地方

3.1 三种调用方式对比​

方法返回类型等待方式适用场景
invoke()AIMessage同步等待完整结果单次请求、简单场景
batch()List[AIMessage]等待所有请求完成批量处理、离线任务
stream()Iterator[AIMessageChunk]流式返回、增量输出聊天机器人、长文本生成

4. 思考内容分离：reasoning_split 参数​

在创建模型时，可以通过 extra_body 设置高级参数：

llm = ChatOpenAI(
    api_key=API_KEY,
    base_url=BASE_URL,
    model=DEFAULT_MODEL,
    extra_body={"reasoning_split": True},  # 将思考内容分离
)

启用后，模型的思考过程会被分离到 reasoning_details 字段，而 content 只包含最终回答：

ai_message = llm.invoke(prompt.invoke({"query": "解释一下为什么天空是蓝色的"}))

print("最终回答:", ai_message.content)
print("思考过程:", ai_message.additional_kwargs.get("reasoning_details"))

这个功能适用于支持思考过程分离的模型（如某些推理模型），方便对思考过程进行审查或展示。

5. 完整示例：结合 Prompt 和 Model​

来看一个完整的例子，串联前面的知识：

import os
from datetime import datetime

import dotenv
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

dotenv.load_dotenv()

# 1. 编排 Prompt
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是{assistant}，现在的时间是{now}。"),
        ("human", "{query}")
    ]
).partial(
    assistant="小助手",
    now=datetime.now
)

# 2. 创建模型
llm = ChatOpenAI(
    api_key=os.getenv("API_KEY"),
    base_url=os.getenv("BASE_URL"),
    model=os.getenv("DEFAULT_MODEL"),
)

# 3. 方式一：单个请求
ai_msg = llm.invoke(prompt.invoke({
    "query": "推荐一道简单的家常菜"
}))
print(f"单个请求: {ai_msg.content}\n")

# 4. 方式二：批量请求
batch_results = llm.batch([
    prompt.invoke({"query": "什么是Python？"}),
    prompt.invoke({"query": "用三个词形容大海"}),
])
for r in batch_results:
    print(f"批量结果: {r.content}")

# 5. 方式三：流式输出
print("\n流式输出: ", end="")
for chunk in llm.stream(prompt.invoke({"query": "说一句话"})):
    print(chunk.content, end="", flush=True)
print()

6. 总结​

功能方法返回类型场景
单次调用invoke()AIMessage简单请求
批量处理batch()List[AIMessage]并行处理多个请求
流式输出stream()Iterator[AIMessageChunk]实时展示生成过程
思考分离extra_body={"reasoning_split": True}reasoning_details推理模型审查

LangChain 1.2 的 Model 组件设计得很简洁，核心就是 invoke、batch、stream 三个方法。掌握这三个方法，基本就能应对大部分场景了。

LangChain：Prompt 组件及使用技巧

2025年12月26日 · 阅读需 10 分钟
GuguLH
Back-end engineers

前言​

Prompt 是大模型应用的核心入口，LangChain 提供了完善的提示模板管理能力。本文基于 LangChain 1.2，围绕官方示例代码，聊聊怎么用好 Prompt 组件。

1. 先跑起来：最基础的用法​

直接看代码：

from langchain_core.prompts import PromptTemplate

prompt = (
    PromptTemplate.from_template("请讲一个关于{subject}的冷笑话")
    + "，让我开心下"
    + "\n使用{language}语言"
)
print(prompt.invoke({"subject": "程序员", "language": "中文"}).to_string())

输出：

请讲一个关于程序员的冷笑话，让我开心下
使用中文语言

几点说明：

  • PromptTemplate.from_template() 是最常用的创建方式，花括号里的 {subject} 是变量占位符
  • 模板可以用 + 号拼接普通字符串，很灵活
  • invoke() 方法接收字典参数，返回 PromptValue 对象
  • to_string() 把结果转成字符串输出

2. format 和 invoke 的区别​

除了 invoke()，还有个 format() 方法：

from langchain_core.prompts import PromptTemplate

prompt = PromptTemplate.from_template("请讲一个关于{subject}的冷笑话")

# 方式一：format 返回字符串
print(prompt.format(subject="喜剧演员"))
# 输出: 请讲一个关于喜剧演员的冷笑话

# 方式二：invoke 返回 PromptValue
prompt_value = prompt.invoke({"subject": "程序员"})
print(prompt_value.to_string())
print(prompt_value.to_messages())

区别：

方法返回类型适用场景
format()str只需要字符串，不需要后续处理
invoke()PromptValue需要转成消息列表、或者链式调用

PromptValue 是 LangChain 内部统一的消息格式，它可以被转换成字符串或消息列表，适配不同的模型接口。

2.1 to_string() 和 to_messages() 详细对比​

invoke() 返回的 PromptValue 有两个转换方法，用法不同。先看字符串模板：

from langchain_core.prompts import PromptTemplate

str_prompt = PromptTemplate.from_template("请讲一个关于{subject}的冷笑话")
str_value = str_prompt.invoke({"subject": "程序员"})

print("=== PromptTemplate ===")
print("to_string():")
print(str_value.to_string())
print()
print("to_messages():")
print(str_value.to_messages())

输出：

=== PromptTemplate ===
to_string():
请讲一个关于程序员的冷笑话

to_messages():
[HumanMessage(content='请讲一个关于程序员的冷笑话')]

再看聊天模板：

from langchain_core.prompts import ChatPromptTemplate

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是{assistant}，当前时间是 {now}"),
    ("human", "我喜欢 {hobby}，请推荐一部相关电影"),
])

chat_value = chat_prompt.invoke({
    "assistant": "小助手",
    "now": "2024年1月",
    "hobby": "编程"
})

print("=== ChatPromptTemplate ===")
print("to_string():")
print(chat_value.to_string())
print()
print("to_messages():")
for msg in chat_value.to_messages():
    print(f"  - {type(msg).__name__}: {msg.content}")

输出：

=== ChatPromptTemplate ===
to_string():
System: 你是小助手，当前时间是 2024年1月
Human: 我喜欢编程，请推荐一部相关电影

to_messages():
  - SystemMessage: 你是小助手，当前时间是 2024年1月
  - HumanMessage: 我喜欢编程，请推荐一部相关电影

两者的核心区别：

方法返回内容返回类型适用场景
to_string()拼接好的纯文本，角色以 "System:" / "Human:" 前缀呈现str传给不支持消息角色的模型，如 text-davinci-003
to_messages()拆成多个消息对象，每个带角色和内容List[BaseMessage]传给支持聊天接口的模型，如 GPT-3.5-turbo、Claude

简单记法：

  • 字符串模板 + to_string() = 纯文本提示
  • 聊天模板 + to_messages() = 结构化消息列表

3. 聊天场景：用 ChatPromptTemplate​

跟聊天模型交互时，消息是有角色区分的。ChatPromptTemplate 就是为此设计的：

from langchain_core.prompts import (
    ChatPromptTemplate,
    HumanMessagePromptTemplate,
    MessagesPlaceholder,
)
from langchain_core.messages import AIMessage
from datetime import datetime

chat_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是OpenAI开发的聊天机器人,请根据用户的提问进行回复,当前时间为:{now}"),
        # 有时候可能还有其他消息,但是不确定
        MessagesPlaceholder("chat_history"),
        HumanMessagePromptTemplate.from_template("请讲一个关于{subject}的冷笑话"),
    ]
).partial(now=datetime.now)

chat_prompt_value = chat_prompt.invoke({
    "chat_history": [
        ("human", "我叫GuguLH"),
        AIMessage("你好,我是ChatGPT,有什么可以帮助您的?")
    ],
    "subject": "程序员",
})
print(chat_prompt_value)
print(chat_prompt_value.to_string())

几个关键点：

  • ChatPromptTemplate.from_messages() 接受元组列表，每个元组是 (角色, 内容) 格式
  • 常用角色：system（系统）、human（用户）、ai（助手）
  • MessagesPlaceholder("chat_history") 是占位符，运行时动态填入聊天历史，这招很实用
  • .partial(now=datetime.now) 可以预填充某些变量，不用每次调用都传。这里传的是函数本身 datetime.now，而不是 datetime.now() 的结果，这样每次调用时都会重新计算当前时间

4. 模板组合：用加号拼接​

多个模板可以拼接成一个，代码更模块化：

from langchain_core.prompts import ChatPromptTemplate

system_chat_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是OpenAI开发的聊天机器人,请根据用户的提问进行回复, 我叫{username}"),
    ]
)

human_chat_prompt = ChatPromptTemplate.from_messages(
    [
        ("human", "{query}")
    ]
)

chat_prompt = system_chat_prompt + human_chat_prompt
print(chat_prompt.invoke({
    "username": "GuguLH",
    "query": "你好,你是?",
}))

system_chat_prompt 定义了系统角色，human_chat_prompt 定义了用户输入，用加号拼在一起就是完整的聊天提示模板。这种写法比把所有东西写在一个大字符串里清晰多了。

5. 模板复用：拆成小块更灵活​

把提示词拆成多个独立模块，再组装成完整模板，好处是便于维护和复用：

from langchain_core.prompts import PromptTemplate

# 描述模板
instruction = "你正在模拟{person}"
# 示例模板
example = """
    下面是一个例子:
    Q:{example_q}
    A:{example_a}
    """

# 开始模板
start = """
    现在你是一个真实的人,请回答用户的问题:
    Q:{input}
    """

full_template = PromptTemplate.from_template(
    f"""
    {instruction}

    {example}

    {start}
    """
)

print(full_template.invoke({
    "person": "雷军",
    "example_q": "你最喜欢的汽车是什么?",
    "example_a": "小米su7",
    "input": "你最喜欢的手机是什么?",
}).to_string())

这个例子把提示词分成三部分：身份设定、示例问答、实际问答。组合时用 f-string 拼成完整模板。实际项目中可以把 instruction、example 这些做成可复用的函数。

6. 两个模板的关系​

记住两条线：

简单记忆：

模板输出
PromptTemplate纯文本字符串 / [HumanMessage]
ChatPromptTemplate带角色前缀文本 / [SystemMessage, HumanMessage, ...]

6.1 ChatPromptTemplate 的内部组成​

from_messages() 接收的列表里，每一项可以是不同类型：

举例：

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是{assistant}"),           # 字符串元组
    SystemMessage(content="固定内容"),       # 消息对象
    HumanMessagePromptTemplate.from_template("我叫{name}"),  # 模板对象
    MessagesPlaceholder("history"),         # 占位符，运行时填入
    "{query}"                               # 字符串，自动转 HumanMessagePromptTemplate
])

6.2 两种输出的对比​

记住这个对应关系：

调用方式场景
PromptTemplate + to_string()简单任务，模型只需要一段文字
PromptTemplate + to_messages()模拟单轮对话
ChatPromptTemplate + to_string()调试时打印查看
ChatPromptTemplate + to_messages()真实聊天模型调用

7. 几个常用的写法​

1. 字符串模板直接拼接

prompt = (
    PromptTemplate.from_template("你是{role}")
    + "，请回答以下问题："
    + "\n{question}"
)

2. 预填充部分变量

base_prompt = ChatPromptTemplate.from_messages([
    ("system", "当前日期是 {date}"),
    ("human", "{query}"),
]).partial(date="2024-01-01")

把固定不变的变量用 .partial() 预填，后续调用时就不用重复传了。

3. 动态聊天历史

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是助手"),
    MessagesPlaceholder("history"),
    ("human", "{input}"),
])

chat_prompt.invoke({
    "history": [("human", "你好"), ("ai", "你好，有什么可以帮你？")],
    "input": "今天天气怎么样",
})

用 MessagesPlaceholder 占住聊天历史的位置，调用时传入实际的消息列表。

向量数据库的配置与使用

2025年12月29日 · 阅读需 16 分钟
GuguLH
Back-end engineers

前言​

在构建RAG（检索增强生成）应用时，向量数据库是不可或缺的核心组件。本文将介绍向量数据库的基本概念、与传统数据库的差异，以及在LangChain中如何配置和使用Faiss、Weaviate以及自定义向量数据库。

1. 向量数据库的介绍与用途​

1.1 什么是向量数据库？​

向量数据库是一种专门用于存储和检索向量数据的数据库。与传统数据库存储结构化数据（如数字、字符串）不同，向量数据库存储的是高维向量——这些向量是由嵌入模型（Embedding Model）将文本、图片等内容转换而成的数值数组。

举个例子，当我们用嵌入模型处理"笨笨是一只猫咪"这句话时，它会生成一个类似这样的向量：

[0.123, -0.456, 0.789, 0.234, -0.567, ...]

这个向量可能包含几百甚至几千个维度，它以数学形式"编码"了这句话的语义信息。语义相近的内容，其向量在空间中的距离也更近。

1.2 向量数据库的用途​

向量数据库在AI应用中有着广泛的使用场景：

语义搜索

传统搜索依赖关键词匹配，用户搜"猫咪"只能找到包含"猫咪"这个词的内容。而向量搜索可以找到语义相关的内容，比如搜"猫咪"也能找到包含"小猫"、"喵星人"的内容，因为它们的向量表示很接近。

RAG（检索增强生成）

这是当前最热门的应用场景。RAG的工作流程是：

  1. 将文档分块并转换为向量存入向量数据库
  2. 用户提问时，将问题也转换为向量
  3. 在向量数据库中检索与问题最相关的文档片段
  4. 将检索结果提供给大模型，生成有据可依的回答

推荐系统

将用户偏好和物品都转换为向量，通过向量相似度计算推荐最匹配的内容。

去重与聚类

通过向量相似度快速识别重复或相似内容，或将大量数据自动分组。

2. 传统数据库与向量数据库的使用差异​

2.1 两种数据库的差异​

数据存储方式不同

特性传统数据库向量数据库
存储内容结构化数据（数字、字符串、日期等）高维向量（浮点数数组）
数据维度固定字段，通常几十个以内高维向量，可达数千维
数据意义数据本身即含义向量是对原始数据的语义编码

查询方式不同

传统数据库使用精确匹配或范围查询。比如：

  • 查找年龄大于18的用户
  • 查找姓名等于"张三"的记录

向量数据库使用相似性查询。比如：

  • 查找与这段文字语义最相近的10条记录
  • 查找与这张图片最相似的其他图片

索引结构不同

传统数据库常用B树、B+树等索引结构，适合精确匹配和范围查询。

向量数据库使用专门的向量索引，如：

  • IVF（倒排文件索引）：将向量空间划分为多个聚类，查询时只搜索最近的几个聚类
  • HNSW（分层可导航小世界图）：构建多层图结构，实现高效近似最近邻搜索
  • PQ（乘积量化）：将向量压缩，减少存储空间和计算量

2.2 相似性搜索算法​

向量数据库的核心能力是快速找到与查询向量最相似的向量。这需要两个关键组件：相似度度量方法和高效搜索算法。

2.2.1 相似度度量方法​

余弦相似度

余弦相似度衡量两个向量方向的相似程度，计算的是向量夹角的余弦值。取值范围是-1到1，值越大表示越相似。

计算公式：

cos⁡(A,B)=A⋅B∥A∥⋅∥B∥\cos(A, B) = \frac{A \cdot B}{\|A\| \cdot \|B\|}cos(A,B)=∥A∥⋅∥B∥A⋅B​

其中：

  • A · B 是向量点积
  • ||A|| 是向量A的模长（范数）

特点：

  • 只关注方向，不关注向量长度
  • 适合文本语义相似度计算
  • 大多数嵌入模型输出的向量需要用余弦相似度

欧几里得距离

欧几里得距离衡量两个向量在空间中的直线距离，类似于我们日常生活中两点间的距离。值越小表示越相似。

计算公式：

d(A,B)=∥A−B∥2=∑i=1n(ai−bi)2d(A, B) = \|A - B\|_2 = \sqrt{\sum_{i=1}^{n} (a_i - b_i)^2}d(A,B)=∥A−B∥2​=i=1∑n​(ai​−bi​)2​

特点：

  • 同时考虑方向和长度
  • 对向量数值敏感
  • 在自定义向量数据库示例中使用的就是这个方法

两种方法的对比

度量方法值范围越大/越小越相似适用场景
余弦相似度-1 到 1越大越相似文本语义、推荐系统
欧几里得距离0 到 ∞越小越相似图像相似度、空间位置

2.2.2 相似性搜索加速计算​

当向量数量达到百万、千万级别时，逐一计算所有向量的相似度会非常慢。向量数据库采用多种技术来加速搜索：

近似最近邻搜索（ANN）

与其精确找到最近的向量，不如快速找到"足够近"的向量，这就是ANN的核心思想。通过牺牲一点精度，换取数量级的速度提升。

聚类索引（IVF）

将所有向量预先分成多个聚类（cluster），每个聚类有一个中心点。查询时：

  1. 找到距离查询向量最近的几个聚类中心
  2. 只在这些聚类中搜索
  3. 大大减少了需要计算的向量数量

图索引（HNSW）

构建一个多层图结构：

  • 底层包含所有向量
  • 上层是稀疏的"快速通道"
  • 查询时从上层快速定位，逐步下沉到底层
  • 类似于"跳表"的思想

量化压缩（PQ）

将高维向量压缩成短编码：

  • 减少内存占用
  • 加快距离计算
  • 代价是精度有所下降

这些技术的组合使用，使得向量数据库能够在毫秒级别完成百万级向量的相似性搜索。

3. 向量数据库的配置与使用​

下面介绍如何在LangChain中配置和使用三种向量数据库：Faiss、Weaviate以及自定义实现。

3.1 Faiss向量数据库​

Faiss是Facebook（Meta）开源的向量检索库，以高效著称，适合本地部署和快速原型开发。

3.1.1 Faiss的特点​

  • 高性能：支持GPU加速，可处理十亿级向量
  • 轻量级：纯本地运行，无需额外服务
  • 支持多种索引：Flat、IVF、HNSW等
  • 适合场景：本地开发、中小规模数据、快速验证

3.1.2 基本使用流程​

第一步：准备嵌入模型

首先需要一个嵌入模型将文本转换为向量。这里使用OpenAI的嵌入模型：

embeddings = OpenAIEmbeddings(
    base_url=BASE_URL,
    api_key=API_KEY,
    model=DEFAULT_MODEL,
)

第二步：创建向量数据库并存入数据

将文本列表转换为向量并存入Faiss：

texts = [
    "笨笨是一只很喜欢睡觉的猫咪",
    "我喜欢在夜晚听音乐，这让我感到放松。",
    ...
]

metadatas = [
    {"page": 1},
    {"page": 2},
    ...
]

# 使用Faiss,需安装faiss-cpu/faiss-gpu

db = FAISS.from_texts(texts, embeddings, metadatas)

第三步：保存到本地

Faiss支持将向量数据库持久化到磁盘：

db.save_local("./vector-store/")

第四步：从本地加载

下次使用时可以直接加载：

db = FAISS.load_local(
    "./vector-store/",
    embeddings,
    allow_dangerous_deserialization=True
)

第五步：执行相似性搜索

results = db.similarity_search_with_score("我养了一只猫,叫笨笨")

similarity_search_with_score返回结果包含相似度分数，分数越小表示越相似。

3.1.3 增删改查操作​

添加新数据

ids = db.add_texts(["笨笨是一只已经8岁的猫了"])

删除数据

db.delete([doc_id])

带过滤条件的搜索

results = db.similarity_search_with_relevance_scores(
    "我养了一只猫,叫笨笨",
    filter=lambda x: x["page"] > 5
)

3.2 Weaviate向量数据库​

Weaviate是一个开源的向量数据库，提供了更完善的生产级特性，如持久化存储、分布式部署、丰富的过滤能力等。

3.2.1 Weaviate的特点​

  • 生产级：支持集群部署、高可用
  • 内置嵌入：可选使用内置的嵌入模型
  • 强大的过滤：支持复杂元数据过滤
  • 多模态：支持文本、图片等多种数据类型
  • GraphQL API：灵活的查询接口

3.2.2 部署方式​

最简单的方式是使用Docker：

docker run -d \
  --name weaviate-dev \
  -p 8080:8080 \
  -p 50051:50051 \
  cr.weaviate.io/semitechnologies/weaviate:1.36.8
  • 8080端口：REST API
  • 50051端口：gRPC API（用于高效的数据传输）

3.2.3 基本使用流程​

第一步：创建连接客户端

# 使用weaviate,需安装langchain-weaviate
client = weaviate.connect_to_local(
    host="127.0.0.1",
    port=8080,
    grpc_port=50051,
)

第二步：创建向量数据库实例

db = WeaviateVectorStore(
    client=client,
    index_name="Dataset",    # 类似于数据库的表名
    text_key="text",          # 存储原始文本的字段名
    embedding=embeddings,
)

第三步：添加数据

ids = db.add_texts(texts, metadatas)

第四步：执行相似性搜索

基本搜索：

results = db.similarity_search_with_score("笨笨")

带过滤条件的搜索：

from weaviate.classes.query import Filter

filters = Filter.by_property("page").less_or_equal(5)
results = db.similarity_search_with_score("笨笨", filters=filters)

第五步：转换为检索器

LangChain的检索器（Retriever）是一个标准接口，方便与链（Chain）集成：

retriever = db.as_retriever()
results = retriever.invoke("笨笨")

3.3 自定义向量数据库​

LangChain提供了VectorStore基类，我们可以继承它实现自己的向量数据库。这对于理解向量数据库的内部原理非常有帮助。

3.3.1 VectorStore基类的核心方法​

要实现一个自定义向量数据库，需要实现以下方法：

方法作用
add_texts()添加文本数据，返回文档ID列表
similarity_search()根据查询文本执行相似性搜索
from_texts()类方法，从文本列表创建向量数据库实例

3.3.2 实现一个基于内存的向量数据库​

下面实现一个简单的内存向量数据库，使用欧几里得距离作为相似度度量：

类定义

class MemoryVectorStore(VectorStore):
    store: Dict = {}  # 存储向量的字典

    def __init__(self, embedding: Embeddings):
        self._embedding = embedding

实现add_texts方法

这个方法负责将文本转换为向量并存储：

def add_texts(self, texts, metadatas=None, *, ids=None):
    # 1. 检查元数据格式
    if metadatas is not None and len(metadatas) != len(texts):
        raise ValueError("metadata格式错误")

    # 2. 将文本转换为向量
    embeddings = self._embedding.embed_documents(texts)

    # 3. 生成唯一ID
    ids = [str(uuid.uuid4()) for _ in texts]

    # 4. 存储每条记录
    for idx, text in enumerate(texts):
        self.store[ids[idx]] = {
            "id": ids[idx],
            "text": text,
            "vector": embeddings[idx],
            "metadatas": metadatas[idx] if metadatas else {}
        }

    return ids

实现similarity_search方法

这个方法负责执行相似性搜索：

def similarity_search(self, query: str, k: int = 4):
    # 1. 将查询转换为向量
    query_embedding = self._embedding.embed_query(query)

    # 2. 计算与所有存储向量的距离
    results = []
    for key, record in self.store.items():
        distance = self._euclidean_distance(
            query_embedding,
            record["vector"]
        )
        results.append({"distance": distance, **record})

    # 3. 按距离排序（距离越小越相似）
    sorted_results = sorted(results, key=lambda x: x["distance"])

    # 4. 返回前k个结果
    top_k = sorted_results[:k]

    return [
        Document(
            page_content=item["text"],
            metadata={**item["metadatas"], "score": item["distance"]}
        )
        for item in top_k
    ]

实现欧几里得距离计算

def _euclidean_distance(self, vec1, vec2):
    return np.linalg.norm(np.array(vec1) - np.array(vec2))

欧几里得距离的计算原理：

  • 计算两个向量在每个维度上的差值
  • 将所有差值平方后求和
  • 对结果开平方根

实现from_texts类方法

@classmethod
def from_texts(cls, texts, embedding, metadatas=None, **kwargs):
    store = cls(embedding=embedding)
    store.add_texts(texts, metadatas, **kwargs)
    return store

3.3.3 使用自定义向量数据库​

# 创建向量数据库实例
db = MemoryVectorStore(embedding=embeddings)

# 添加数据
ids = db.add_texts(texts, meta_datas)

# 执行搜索
results = db.similarity_search("笨笨")

4. 总结​

向量数据库是现代AI应用的基础设施，理解其原理和使用方式对于构建RAG等应用至关重要。

核心要点回顾：

  1. 向量数据库存储的是语义向量，支持相似性搜索而非精确匹配
  2. 相似度度量有余弦相似度和欧几里得距离两种主要方法
  3. 加速搜索依赖ANN、聚类索引、图索引等技术
  4. LangChain提供了统一的VectorStore接口，支持Faiss、Weaviate等多种后端
  5. 自定义向量数据库可以帮助理解原理，但生产环境建议使用成熟的方案

选型建议：

  • 学习和原型验证：Faiss或自定义实现
  • 中小规模生产：Faiss（本地）或Weaviate（单机）
  • 大规模生产：Weaviate集群或其他专业向量数据库（如Milvus、Pinecone）
